Project Idea & Vision

  • Deployment: Docker, Frontend + BFF in the same container

Security-Relevant:

The web application is publicly accessible ‚Äî however, the customer‚Äôs warehouse management software is only locally reachable. Synchronization happens indirectly.

First Experiences with Claude Code

The first steps were promising:

  • The basic structure of frontend & backend was neatly generated
  • Docker support and mock data were automatically created
  • A prototype with a working UI was created in a very short time

But: The more specific the requirements became, the more frequently Claude Code stumbled.

Examples:

  • Changes to the appearance broke the working navigation
  • The authentication flow (OAuth2) was never correctly implemented
  • Problems that were already solved reappeared in later iterations
  • Security-critical concepts such as token handling were violated

Personal Conclusion

Currently, there are numerous reports on the internet about so-called ‚ÄúVibe Coding,‚Äù where AI is celebrated as a game-changer.

Often accompanied by the question: Do we even need software developers in the future?

My short answer: Sort of. Yes, it is now indeed possible to build working applications without deep programming knowledge. But: Developers ‚Äî especially experienced ones ‚Äî are more necessary than ever.

The long answer: Working with coding agents reminds me of Schmendrick the Magician from *The Last Unicorn*. They work best when you give them as much freedom as possible. I quote: ‚ÄúMagic ‚Äî do what you will!" Then the results are sometimes impressive, fast, and surprisingly effective. But as soon as the requirements become specific and complex, the magic runs into its limits: The agents need close control. Reviews become mandatory. And it‚Äôs often faster to write code directly than to describe the requirements in such precision that an agent understands them correctly.

However, the prerequisite is that you can actually recognize where the problems occur. An amateur often doesn‚Äôt. I only changed code manually when the agent was stuck in an endless loop. And even now, the code still contains numerous open issues ‚Äî without tests, with messy patterns, and unclear responsibilities. Would you trust such a system with your credit card details?

The role of AI here is ambivalent: It relieves repetitive tasks. But it creates new control and review needs. And it can ‚Äî if used incorrectly ‚Äî even increase risks.
